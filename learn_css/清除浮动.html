清除浮动是老生常谈的问题，在网页布局的时候，能够通过几种方法尽数解决，但我知道，在这个过程中，我对于其原理是十分含糊的，今天看到一丝的[这片文章](http://www.iyunlu.com/view/css-xhtml/55.html)理了一下。

什么是浮动？

浮动指的是给一个元素设置类浮动属性后，该元素会根据设置的方向，向左或者向右进行，移动，直到它碰到父元素或者另一个浮动元素的边缘。也就是说，元素会脱离我们常讲的文档流(normal flow).

父元素塌陷

一个元素浮动之后，因为浮动元素脱离文档流，文档中的元素会视其不存在，继续向上排列，如果浮动元素的高度大于包含元素的内部高度时，就会如下所示的现象，当然父元素中没有其他元素的时候，它会塌陷到什么都不剩~

float:left;
糟糕，我塌陷了。

行内元素浮动之后会发生什么？

我们知道，在行内元素上设置width/height等属性之后，不会起作用，但是，那天我冷不丁在设置了高度和宽度的span元素上，加了个float，结果，width和height竟然起作用了！伙惊呆~

经过一番研究，原来，当行内元素设置了浮动属性后，其会具有块级元素的特征，相当于display: block;

清楚浮动与关闭浮动？

一丝的文章中，强调了清楚浮动和关闭浮动是不一样的，清除浮动是指对元素应用clear: both;样式，而闭合浮动指的是将浮动“关起来”，不要影响后面的元素。

清除浮动的三种方法

  1. 添加额外标签再设置clear清除
  2. 使用:after伪元素clear清除
  3. 父元素设置overflow: hidden;关闭浮动
  4. 父元素也设置浮动
  5. 父元素设置display: table;
